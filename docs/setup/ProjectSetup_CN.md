### **核心理念："模块化和可验证"**

我们将以不同的、模块化的阶段构建这个项目。每个阶段的目标是生成您可以在进入下一个阶段之前可视化和验证的模型或数据集。这可以防止您在单一的、整体的流程中迷失方向。

### **第 0 阶段：设置、探索和可视化**

**目标：** 深入了解数据并建立一个强大的环境。

**实施步骤：**

1. **环境设置：**
    * 安装必要的库：`pandas`、`numpy`、`pydicom`、`nibabel`（用于 NIfTI 掩码）、`scikit-image`、`matplotlib`、`plotly` 或 `itkwidgets`（用于 3D 可视化）。
    * 安装机器学习库：`pytorch` 或 `tensorflow`。
    * **强烈推荐：** 安装医学影像库，如 **MONAI**（基于 PyTorch）或 `torchio`。它们有用于重采样、3D 增强和 3D 模型架构（如 U-Net）的预构建工具，将为您节省数百小时。

2. **初始数据探索（EDA）：**
    * 加载 `train.csv`。分析 `Modality`、`PatientSex` 和 14 个目标标签的分布。
    * 加载 `train_localizers.csv`。分析动脉瘤在 13 种位置类型中的分布。
    * 合并这两个数据框，将动脉瘤位置与患者人口统计数据联系起来。

3. **构建您的可视化工具包（至关重要）：**
    * **函数 1：`view_slice(series_path, slice_index)`：** 一个简单的函数，接受序列路径和切片编号，读取 DICOM，应用正确的特定模态预处理（CT 的 HU 窗口化，MR 的百分位数标准化），并使用 `matplotlib` 显示。
    * **函数 2：`view_slice_with_overlay(series_path, slice_index, localizer_df, mask_path)`：** 这是您最重要的工具。它应该：
        1. 显示预处理后的 DICOM 切片。
        2. 如果该切片上有动脉瘤（检查 `localizer_df`），将其 `(x, y)` 坐标绘制为图像上的亮红点。
        3. 如果存在分割掩码（`mask_path`），从 `.nii` 文件加载对应的切片，并将其透明地覆盖在 DICOM 图像上。使用颜色地图显示不同的血管标签。

**可视化指导：**

* 查看有动脉瘤的扫描时，使用叠加函数同时查看动脉瘤点和血管掩码。这将让您直观地感受到数据的结构。**您应该能够直观地确认红点位于彩色血管上。**

### **第 1 阶段：血管分割模型（子问题）**

**目标：** 创建一个可以为*任何*输入扫描生成血管掩码的模型。

**实施步骤：**

1. **数据集：**
    * 识别具有分割掩码的 **178 个 `SeriesInstanceUID`**。这些是您的训练示例。
    * 输入（`X`）：3D DICOM 扫描。
    * 目标（`Y`）：3D `.nii` 血管掩码（不是 `_cowseg.nii` 那种）。

2. **预处理管道（可在所有阶段重用）：**
    * 创建一个强大的预处理函数，接受序列路径和掩码路径。
    * 它应该执行通用步骤：加载 -> 排序 -> **特定模态标准化** -> 重采样到各向同性 `1mm` 间距 -> 重新定向到 RAS -> 填充/裁剪到固定大小（如 `128x192x192`）。
    * 对扫描及其掩码应用此流程，以确保它们保持完美对齐。

3. **模型架构：**
    * 使用 **3D U-Net**。不要从头构建。使用经过实战检验的实现，如 **MONAI** 中的（`monai.networks.nets.UNet`）。

4. **训练：**
    * **损失函数：** 使用 Dice 损失和交叉熵损失的组合（MONAI 中的 `DiceCELoss`）。这对于分割任务非常出色。
    * **数据增强：** 使用 `torchio` 或 `monai.transforms` 中的 3D 特定增强：随机翻转、旋转、弹性变形。
    * **验证：** 将 178 个样本分为训练和验证集（如 150 个训练，28 个验证）。监控验证 Dice 分数以防止过拟合。

**可视化指导：**
* 训练后，从验证集中取一个扫描。
* 从您的模型生成预测。
* 并排显示三张图像：**(1) 原始扫描切片，(2) 真实标签掩码切片，(3) 您模型的预测掩码切片。** 这将立即显示您的模型学习血管解剖学的效果如何。

### **第 2 阶段：动脉瘤定位模型（候选生成）**

**目标：** 创建一个可以生成可能的动脉瘤位置"热图"的模型。

**实施步骤：**

1. **数据集：**
    * 使用至少有一个动脉瘤的 **1,890 次扫描**。
    * **输入（`X`）：** 这应该是**双通道 3D 输入**。
        * 通道 1：预处理后的扫描。
        * 通道 2：您从第 1 阶段模型生成的**预测血管掩码**。
    * **目标（`Y`）：** 为每个扫描创建 3D 真实热图。这是一个零数组，在 `train_localizers.csv` 中每个动脉瘤坐标的中心放置一个小的、明亮的 3D 高斯球体。

2. **模型架构：**
    * 另一个 **3D U-Net** 非常适合此任务。它将接受双通道输入并输出单通道热图。

3. **训练：**
    * **损失函数：** 如果您将其视为回归问题，均方误差（MSE）是一个很好的起点。或者，您可以将其视为分割任务并再次使用 Dice 损失。
    * 使用相同的数据增强流程。

**可视化指导：**
* 从验证集中取一个阳性扫描。
* 运行您的模型以获得预测的热图。
* 将此热图（使用高透明度的"hot"或"jet"颜色地图）叠加到原始扫描切片上。**您应该看到模型预测的亮点直接位于真实动脉瘤位置的中心。**

### **第 3 阶段：基于补丁的分类器（候选验证）**

**目标：** 创建一个高度准确的分类器，可以区分小 3D 补丁中的真实动脉瘤和"模拟物"。

**实施步骤：**

1. **数据集创建（最重要的策略步骤）：**
    * **正补丁（N=2,286）：** 对于 `train_localizers.csv` 中的每个坐标，从预处理后的扫描中提取以该坐标为中心的小 3D 补丁（如 `48x48x48` 像素）。将这些标记为 **1**。
    * **硬负补丁（N ≈ 5,000-10,000）：** 在 **2,515 次无动脉瘤扫描**上运行您的第 2 阶段模型。识别其产生最高假阳性预测的位置。提取以这些"模拟物"位置为中心的补丁。将这些标记为 **0**。
    * **简单负补丁（N ≈ 2,000）：** 从阴性扫描中的健康组织中提取一些随机补丁。将这些标记为 **0**。
    * 将这些合并到您的最终训练集中。

2. **模型架构：**
    * 3D CNN 分类器。ResNet 的 3D 版本（如 `ResNet3D`）或带有几个卷积层后跟分类器头的定制 CNN 会很好地工作。输入大小很小，因此模型不需要很大。

3. **训练：**
    * **损失函数：** 标准二分类交叉熵（BCE）。
    * **数据增强：** 对补丁应用 3D 增强。

**可视化指导：**
* 查看几十个正补丁和硬负补丁。尝试查看您是否能发现差异。这将帮助您理解任务的难度以及您的模型可能正在学习的特征。

### **第 4 阶段：最终推理和提交**

**目标：** 将所有训练好的模型组合成一个单一的流程，生成提交文件。

**实施步骤（在 `predict` 函数内部）：**

1. **加载：** 加载测试 DICOM 序列。
2. **预处理：** 应用您完整的、可重用的预处理流程。
3. **生成血管掩码（第 1 阶段模型）：** 将预处理后的扫描输入到您的血管分割 U-Net 中，以获得预测的血管掩码。
4. **生成热图（第 2 阶段模型）：** 将双通道（扫描 + 掩码）体积输入到您的定位 U-Net 中，以获得热图。
5. **识别候选：** 找到热图中的前 N 个峰值。
6. **提取补丁：** 提取围绕这些候选坐标的小 3D 补丁。
7. **分类补丁（第 3 阶段模型）：** 将每个补丁输入到您的 3D 分类器中，以获得每个候选的概率分数（0 到 1 之间）。
8. **聚合与提交：**
    * **"动脉瘤存在"分数：** 整个扫描的最终分数是其所有分类补丁的**最大概率**。
    * **特定位置分数：** 找到得分最高的补丁。查找该补丁中心坐标处预测血管掩码的值。如果掩码值为 `9`，则为"右侧大脑中动脉"列分配高概率，而对其他 12 列分配低概率。
    * 将这 14 个概率格式化为所需的 DataFrame。
