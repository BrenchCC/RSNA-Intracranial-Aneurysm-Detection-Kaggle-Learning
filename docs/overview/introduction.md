# 一、竞赛基本信息（你首先要能讲清楚）

**竞赛名称**
RSNA Intracranial Aneurysm Detection

**举办方**

* Radiological Society of North America（RSNA）

**竞赛目标**

利用深度学习模型，在医学影像中：

1. **检测是否存在脑动脉瘤**
2. **定位动脉瘤的位置**

换句话说就是：

> 输入：脑部医学影像
> 输出：是否存在动脉瘤 + 动脉瘤在哪

这个任务属于典型的：

* **医学影像检测任务**
* **病灶定位任务**

该挑战旨在开发 AI 工具辅助放射科医生，提高动脉瘤的检测效率和准确性。([rsna.org][1])

---

# 二、医学背景（面试时一定会问）

## 什么是脑动脉瘤

**Intracranial Aneurysm**

是脑血管壁局部变薄、鼓起形成的小囊。

危险点：

* 破裂会导致 **蛛网膜下腔出血**
* 死亡率很高

统计：

* 全球约 **3% 人群有动脉瘤**
* **约50%在破裂后才被发现** ([rsna.org][2])

所以：

**早期发现非常重要**

AI 的价值：

* 自动筛查
* 减少医生漏诊

# 三、数据集是什么

这个竞赛的数据非常典型的 **医学影像数据**。

## 数据来源

来自：

* **18家医疗机构**
* **5个大洲**
* **60多名放射科专家标注** ([rsna.org][1])

这点面试可以强调：

> 数据多中心（multi-center），更接近真实临床环境。

---

## 数据模态（Imaging modalities）

包含多种影像：

| 模态  | 含义     |
| --- | ------ |
| CTA | CT血管造影 |
| MRA | MR血管造影 |
| MRI | 磁共振    |

这些都是：

**3D医学影像**

---

## 数据结构

医学影像不是普通图片，而是：

**3D体数据**

结构一般是：

```
Study
   ├── Series
   │      ├── slice_001
   │      ├── slice_002
   │      ├── slice_003
```

可以理解为：

**一个病例 = 一堆连续切片**

---

## 数据规模

公开讨论中常提到规模约：

* **≈ 4000 个病例**

但每个病例：

* 都是 **3D扫描**

所以数据量其实很大。

# 四、任务定义（核心）

模型需要预测：

### 1 是否存在动脉瘤

```
Aneurysm Present
```

二分类：

```
0 = 没有
1 = 有
```

---

### 2 动脉瘤具体位置

脑动脉有 **13个常见位置**。

模型需要预测：

```
13个位置
是否存在动脉瘤
```

例如：

| 血管位置 | 是否有动脉瘤 |
| ---- | ------ |
| ACA  | 0      |
| MCA  | 1      |
| ICA  | 0      |
| ...  | ...    |

---

所以最终输出：

```
14个概率
```

即：

```
1个：Aneurysm Present
13个：各血管位置
```

---

# 五、评价指标（非常关键）

这个竞赛的评分方式很特殊。

### 每个样本预测 14 个概率

然后计算：

**加权 LogLoss**

---

## 权重分布

| 任务               | 权重     |
| ---------------- | ------ |
| Aneurysm Present | **13** |
| 每个位置             | 1      |

也就是说：

```
总权重 = 26
```

---

## 这意味着什么

这个指标设计会导致一个策略：

**优先保证检测到是否存在动脉瘤**

因为：

```
漏检动脉瘤 → 惩罚非常重
```

因此：

很多方法会先做：

```
是否有动脉瘤
```

再做：

```
具体在哪
```

---

# 六、为什么这个任务很难

医学影像检测困难主要有几个原因：

## 1 病灶非常小

动脉瘤通常：

```
2mm - 10mm
```

在 CT 中只占很小区域。

---

## 2 数据极度不平衡

大多数扫描：

```
没有动脉瘤
```

即：

```
positive << negative
```

---

## 3 3D数据

普通目标检测：

```
2D图片
```

医学影像：

```
3D体数据
```

模型复杂度更高。

---

## 4 不同设备扫描

不同医院：

* CT机器不同
* 参数不同

这导致：

**Domain Shift**

---

# 七、典型解决方案（Kaggle常见思路）

几乎所有高分方案都用：

## 多阶段 pipeline

```
Stage1
血管分割

Stage2
ROI提取

Stage3
分类 / 检测
```

---

## Stage1：血管分割

目的：

先找出 **脑血管区域**

因为动脉瘤只会出现在血管上。

常用模型：

```
3D U-Net
nnUNet
```

---

## Stage2：ROI检测

ROI = Region of Interest

只保留：

```
血管附近区域
```

好处：

* 减少背景
* 提升模型效率

---

## Stage3：分类

在 ROI 上预测：

```
是否有动脉瘤
在哪个血管
```

常用：

```
3D CNN
2.5D CNN
Transformer
```

---

# 九、你在面试时可以这样介绍这个项目

建议用这种结构：

### 1 研究背景

> 脑动脉瘤是一种严重脑血管疾病，破裂可能导致致命出血，因此早期检测非常重要。

---

### 2 研究目标

> 本项目基于RSNA脑动脉瘤检测竞赛数据，利用深度学习方法自动检测脑动脉瘤并定位其在13个血管位置中的具体位置。

---

### 3 数据

> 数据包含来自18个医疗机构的CT和MR影像，由60多名放射科专家标注。

---

### 4 方法

> 我们构建多阶段医学影像分析pipeline，包括血管区域提取、候选区域检测和最终分类。

---

### 5 评价指标

> 竞赛采用加权LogLoss，其中“是否存在动脉瘤”的权重为13，因此模型需要优先保证高灵敏度。
